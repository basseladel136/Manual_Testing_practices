# [TodoApp](https://github.com/basseladel136/todoapp) testing
In this project I tried to apply my skills in manual testing by:
- [x] exploring the app.
- [x] analyzing it.
- [x] design and implement tests.

## Work products
- [Test Cases](PASTE_YOUR_GOOGLE_SHEET_LINK_HERE) — 56 manual test cases covering Signup, Login, and the Todo page.
- [Bug Reports](PASTE_YOUR_GOOGLE_SHEET_LINK_HERE).

## Test levels:
- System Testing.
- User Acceptance Testing (UAT).

## Types of performed tests
+ Functional testing.
+ Black-box testing using:
   - Equivalence Partitioning (EP)
   - Boundary Value Analysis (BVA).
   - Decision table.
   - State transition techniques.
+ Usability testing.

## Tools and extensions used:
* Google Sheets.
* Developer tools (inspect).

## Modules covered
- **Signup** (20 test cases): field validation, password policy, duplicate email, navigation.
- **Login** (12 test cases): field validation, invalid credentials, session behavior, navigation.
- **Todo Page** (24 test cases): add/edit/delete todos, complete/uncomplete, search, status filters, pagination, error states.
